# Features

Um documento por feature, criado com `/create-feature` a partir de
[`templates/feature-template.md`](../../templates/feature-template.md). Uma feature deve ter critérios de aceitação concretos e
testáveis antes de avançar para a arquitetura (`/approve-architecture`).

| _nenhum ainda_ | — | — | — |

> Fluxo: feature → arquitetura → casos de uso → testes → PR.
